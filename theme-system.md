### 1.1 三层颜色模型

| 层级 | 名称 | 说明 | 跨主题变化 |
|------|------|------|-----------|
| **L1 基础色** | Background / Text / Border | 背景层次、文字层级、边框 | ✅ 深浅两套完全不同 |
| **L2 语义色** | Accent / Success / Warning / Error | 功能含义色，不随主题变色相 | ⚠️ 亮度适配 |
| **L3 标注色** | Annotation Palette | 用户标注用色（画笔/箭头/文字等） | ❌ 固定不变 |

## 二、完整颜色令牌定义

### 2.1 深色主题 Dark（默认）

```python
class _DarkTheme:
    # ===== L1 基础色：背景层次（从暗到亮） =====
    BG_CANVAS   = "#0f0f1a"   # 最底层：画布/全屏遮罩底色
    BG_APP      = "#141422"   # 应用主背景：窗口底色
    BG_CARD     = "#1e1e32"   # 卡片/面板背景
    BG_ELEV     = "#282844"   # 悬浮层：输入框、下拉菜单
    BG_HOVER    = "#333358"   # 悬停态：按钮/列表项 hover
    BG_INPUT    = "#181828"   # 输入框专用背景

    # ===== L1 基础色：文字层次（从亮到暗） =====
    TXT_PRIMARY = "#f0f0f8"   # 主文字：标题、重要信息
    TXT_SECONDARY = "#b0b0c8" # 次文字：常规内容
    TXT_TERTIARY = "#707088"  # 辅助文字：提示、说明
    TXT_MUTED   = "#505068"   # 弱化文字：占位符、禁用态

    # ===== L1 基础色：边框层次 =====
    BORDER      = "#2a2a44"   # 默认边框：分割线
    BORDER_STRONG = "#3a3a58" # 强调边框：输入框 focus、活跃态

    # ===== L2 语义色：功能色（深色主题专用值） =====
    ACCENT      = "#4a8cff"   # 主强调色：主操作按钮、选中态、链接
    ACCENT_LIGHT = "#5c9cff"  # 强调色浅变体：hover 态
    ACCENT_BG   = "#1a2848"   # 强调色淡背景：选中项背景、badge

    SUCCESS     = "#4caf84"   # 成功态：保存成功、热键就绪
    WARNING     = "#e09050"   # 警告态：实验性功能、注意
    ERROR       = "#e05555"   # 错误态：删除操作、错误提示
    ERROR_LIGHT = "#f06060"   # 错误色浅变体：danger 按钮 hover

    INFO        = "#4a8cff"   # 信息提示（同 accent）

    # ===== L2 语义色：特殊用途 =====
    CANVAS_BG   = "#10101c"   # 编辑器画布底色（比 app 更暗）
    SELECTOR    = "#4a8cff"   # 选区边框（同 accent）
    MASK_DARK   = "rgba(0,0,0,0.45)"  # 选区外遮罩（深色）

    # ===== L3 视觉效果 =====
    SHADOW      = "0 8px 32px rgba(0,0,0,0.4)"
    SHADOW_LG   = "0 14px 40px rgba(0,0,0,0.45)"
    FOCUS_RING  = "0 0 0 3px rgba(74,140,255,0.18)"
    SEL_FILL    = "rgba(74,140,255,0.06)"
```

### 2.3 标注色板 Annotation Palette（双主题固定不变）

**12 色标准色板** — 用于画笔、箭头、矩形、文字标注等用户创作工具。

| 序号 | 色值 | 名称 | 使用场景 |
|------|------|------|---------|
| 1 | `#FF3B30` | 红色 | 重点标记、删除线、错误标注 |
| 2 | `#FF6B35` | 橙红 | 暖色强调 |
| 3 | `#FF9500` | 橙色 | 警告标注、高亮替代 |
| 4 | `#FFCC02` | 黄色 | 高亮标注、重点区域 |
| 5 | `#34C759` | 绿色 | 通过标记、正确标注 |
| 6 | `#30B0C7` | 青色 | 信息标注 |
| 7 | `#007AFF` | 蓝色 | 链接标注、默认标注色 |
| 8 | `#5856D6` | 紫色 | 特殊强调 |
| 9 | `#AF52DE` | 品红 | 高亮强调 |
| 10 | `#FF2D55` | 玫红 | 极强强调 |
| 11 | `#8E8E93` | 灰色 | 弱化标注、次要内容 |
| 12 | `#FFFFFF` | 白色 | 深色背景上的浅色标注 |

**设计决策**：标注色板在双主题下保持不变。原因：
- 用户创作的标注需要在不同背景下保持视觉一致性
- 截图可能分享给他人，颜色不应因本地主题设置而改变
- 类似 iOS/Windows 系统标注工具的设计惯例

## 三、颜色使用场景规范

### 3.1 背景色使用矩阵

| 组件/区域 | 深色主题 | 浅色主题 | 令牌名 |
|----------|---------|---------|--------|
| 全屏选区遮罩底 | `#0f0f1a` | `#f8f9fb` | `BG_CANVAS` |
| 主窗口背景 | `#141422` | `#ffffff` | `BG_APP` |
| 侧边栏/面板 | `#1e1e32` | `#ffffff` | `BG_CARD` |
| 顶栏/底栏 | `#1e1e32` | `#ffffff` | `BG_CARD` → 实际用 `BG2` |
| 输入框/下拉菜单 | `#282844` | `#f4f6f9` | `BG_ELEV` |
| 按钮 hover | `#333358` | `#eef1f6` | `BG_HOVER` |
| 选中项背景 | `#1a2848` | `#eaf1fe` | `ACCENT_BG` |
| 编辑器画布 | `#10101c` | `#eef0f4` | `CANVAS_BG` |

### 3.2 文字色使用矩阵

| 文字类型 | 深色主题 | 浅色主题 | 对比度(Dark) | 对比度(Light) |
|---------|---------|---------|-------------|--------------|
| 窗口标题 | `#f0f0f8` | `#1a2233` | 16.8:1 ✓ | 15.3:1 ✓ |
| 面板标题 | `#f0f0f8` | `#1a2233` | 同上 | 同上 |
| 常规正文 | `#b0b0c8` | `#4a5568` | 8.2:1 ✓ | 5.9:1 ✓ |
| 辅助提示 | `#707088` | `#8a94a6` | 4.8:1 ✓ | 3.8:1 ✓ |
| 禁用/占位 | `#505068` | `#b0b8c4` | 3.2:1 ⚠ | 2.9:1 ⚠ |

> ⚠ 注：禁用色对比度为 3:1 左右，仅用于大文字（≥18px）或临时性提示

### 3.3 语义色使用规则

#### 主强调色 ACCENT 的使用场景（必须严格遵守）

| ✅ 应该使用 | ❌ 不应使用 |
|------------|------------|
| 主操作按钮（保存、确认、截图） | 大面积背景填充（太抢眼） |
| 当前选中工具高亮 | 错误/危险操作（应用 ERROR 红） |
| 选区边框、尺寸标签 | 成功提示（应用 SUCCESS 绿） |
| 链接、可点击元素 | 作为普通装饰色 |

#### 成功/警告/错误色的使用场景

| 颜色 | 使用场景 | 典型组件 |
|------|---------|---------|
| `SUCCESS` 绿 | 操作成功、正常状态 | "已保存"、"热键就绪"、在线指示器 |
| `WARNING` 橙 | 注意事项、实验性功能 | "滚动截图（实验性）"、提示信息 |
| `ERROR` 红 | 破坏性操作、错误状态 | 删除按钮、错误消息、"取消"按钮 hover |

### 3.4 边框使用规则

| 边框类型 | 色值 | 线宽 | 使用场景 |
|---------|------|------|---------|
| 默认分割线 | `BORDER` | 1px | 面板之间、顶栏/底栏分隔 |
| 输入框边框 | `BORDER` | 1px | 输入框默认态 |
| Focus 边框 | `ACCENT` 色 | 1-2px + `FOCUS_RING` | 输入框聚焦、键盘导航 |
| 强调描边 | `BORDER_STRONG` | 1px | 弹窗、悬浮工具栏 |
| 选区边框 | `SELECTOR`(=ACCENT) | 1.5-2px | 截图选择区域 |

## 五、一致性检查清单

### 5.1 新增组件时的检查项

当开发者新增一个 UI 组件时，必须确认以下每一项：

- [ ] **背景色来源正确**：是否使用了正确的 `T.BGX` 令牌？是否混用了硬编码颜色？
- [ ] **文字色可读**：在该背景色上，使用的文字色对比度 ≥ 4.5:1？
- [ ] **边框可见**：边框色与背景色是否有足够区分？
- [ ] **Hover/Focus 态完整**：鼠标悬停和键盘聚焦时颜色是否明确变化？
- [ ] **双主题验证**：在深色和浅色主题下分别测试过视觉效果？
- [ ] **语义色准确**：是否误用了强调色表示警告/错误？

### 5.2 常见违规模式（禁止）

| 违规模式 | 正确做法 |
|---------|---------|
| 硬编码 `"#ffffff"` 或 `"#000000"` 作为背景 | 使用 `T.BG_APP` / `T.TXT_PRIMARY` 等 |
| 在深色主题下直接写白色文字 `fill="#fff"` | 使用 `T.TXT0`（浅色主题自动变为近黑） |
| 用 ACCENT 蓌色做删除/错误按钮 | 使用 `T.ERROR` / `T.RED` |
| 新增的颜色值未加入本规范 | 必须在此文档中注册新颜色并说明用途 |
| 只测试单一主题 | 双主题都必须测试，确保无残留或冲突样式 |

### 5.3 主题切换完整性验证

切换主题后必须逐项确认：

1. **窗口背景** — 外层 Frame/Canvas 是否更新？
2. **顶栏/底栏** — 背景色、文字色、图标色是否全部刷新？
3. **侧边栏** — 工具按钮的普通态/选中态/hover 态是否正确？
4. **画布区** — Canvas 背景色是否更新？
5. **所有按钮** — StyledButton 是否调用 `_draw()` 重绘？
6. **所有 Canvas 组件** — 颜色圆点、粗细预览、自定义颜色按钮是否重绘？
7. **对话框** — 文字对话框、水印对话框、设置对话框是否跟随主题？
8. **历史记录** — 列表项背景、悬停效果是否正确？
9. **状态栏** — 文字色是否更新？
10. **主题图标** — 太阳/月亮图标是否切换？

## 六、实现指南

### 6.1 Python 端：如何正确使用主题色

```python
# ✅ 正确：通过 T 代理类访问
frame = tk.Frame(parent, bg=T.BG2)           # 面板背景
label = tk.Label(frame, text="标题", fg=T.TXT0, bg=T.BG2)  # 主文字
button_bg = T.ACCENT if is_primary else T.BG3  # 条件取色
border_color = T.BD                               # 边框

# ❌ 错误：硬编码颜色
frame = tk.Frame(parent, bg="#141422")      # 切换到 light 主题后不会变！
label = tk.Label(frame, text="标题", fill="#ffffff")  # 浅色主题下不可读！

# ✅ 正确：动态获取当前主题（适用于运行时计算）
current_accent = T.ACCENT  # 总是返回当前激活主题的值
```

### 6.2 主题切换后的刷新机制

Tkinter 不支持 CSS 式的变量级联刷新，因此采用**追踪+重绘**机制：

```python
# 1) 创建 widget 时注册到追踪列表
self._track(widget, 'bg', 'sidebar')

# 2) 主题切换时统一刷新
def toggle_theme(self):
    T.toggle_theme()
    self._apply_theme_to_tracked()  # 遍历所有已追踪 widget 并更新

# 3) Canvas 类组件需提供重绘方法
def _draw_custom_color_btn(self):
    self._custom_color_btn.delete("all")
    self._custom_color_btn.create_oval(..., outline=T.BD2)  # 用当前 T.* 值
```

### 6.3 HTML 端：CSS 变量的使用

```css
/* ✅ 正确：全部通过 CSS 变量引用 */
.my-component {
  background: var(--bg-card);
  color: var(--txt-0);
  border: 1px solid var(--bd);
}
.my-component:hover {
  background: var(--bg-hover);
}

/* 主题切换只需改 data-theme 属性 */
html[data-theme="dark"] { --accent: #4a8cff; }
html[data-theme="light"] { --accent: #2e6fe0; }
```
